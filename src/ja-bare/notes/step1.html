bare Step 1. init bare

Part 1 に引き続き、こんどは複数のリポジトリがどのようにオブジェクトをやり取りするかを見てみます。

次のステップに進むときは、このコラムの一番下にある 次へ のリンクを使ってください。リポジトリ間を移動することがあります。

画面上部 (黒いヘッダ部分) にある « Prev と Next » は、同じリポジトリ内での変化を見たいときに使います。

$ git init --bare

--bare をつけてリポジトリを作成しました。これをベアリポジトリと呼びます。

このコラムの一番上にある bare は、ベアリポジトリの中身を表示していることを表しています。

ベアリポジトリには作業ディレクトリがありません。.git ディレクトリの中身だけが生成されます。

通常、コミットを push する先のリポジトリは、ベアリポジトリである必要があります。作業ディレクトリがあると、外からの push で HEAD と index の間にズレが生じて不都合だからです。

Github などに、このベアリポジトリが置いてあるものと思えば OK です。

次に、このベアリポジトリを clone します。次へ をクリックして、repo1 リポジトリに移動してください。
