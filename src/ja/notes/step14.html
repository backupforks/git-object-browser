Step 14. fetch

$ git fetch gist

step13 で追加した gist リモートから fetch しました。

config の設定により、gist リモートの refs/heads/master リファレンスが、ローカルの refs/remotes/gist/master にコピーされました。

このように refs/remotes 以下にできるリファレンス(ブランチ)のことを、追跡ブランチ (またはリモートブランチ) と呼びます。追跡ブランチは、リモートのミラーです。追跡ブランチに直接コミットすることはできません。リモートリポジトリでの変更が追跡ブランチに忠実に反映されます。

fetch によって、gist から 3つのオブジェクトをダウンロードしました。

commit オブジェクト、tree オブジェクト、blob オブジェクト の3つです。リモートから fetch したオブジェクトと、ローカルで作ったオブジェクトには何の区別もないことに注目してください。

また、今回 fetch した commit オブジェクトには parent が無いことにも注目です。このコミットは、今までローカルで作ったどのコミットとも親子関係がありません。

Git では、1つのリポジトリの中に複数の独立したコミットツリーを持つことができます。ローカルで親子関係がないコミットを新たに作成したい時は、git checkout コマンドに --orphan (孤児) オプションを指定します。
